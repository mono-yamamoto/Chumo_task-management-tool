このスキルは、difitコマンドを使ってユーザーにコードレビューを依頼する。
ユーザーがレビューコメントを残した場合、difitコマンド終了時にstdoutに出力される。
レビューコメントが返されたら、その指摘に対応して修正を続ける。
コメントなしでサーバーが終了された場合は「レビューコメントなし」として扱う。

## When to use

- ユーザーが「レビューして」「diff見せて」「変更確認して」と言った時
- 実装完了後にユーザーへのレビュー依頼として
- コミット前の変更確認として

## 実行方法

**必ず `run_in_background: true` でバックグラウンド実行すること。**
difitサーバーはユーザーがレビューを終えるまで起動し続けるため、フォアグラウンドだとタイムアウトする。

```bash
# 未コミット変更をレビュー（untracked含む）
difit . --include-untracked

# 最新コミットをレビュー
difit

# ブランチ比較
difit @ main
```

バックグラウンドで起動後、ユーザーにブラウザでレビューしてもらう旨を伝える。
サーバー終了後にTaskOutputでコメントを確認し、指摘があれば修正を行う。

## Workflow

1. `difit . --include-untracked` を **`run_in_background: true`** で実行
2. ユーザーに「ブラウザでdiffが開くのでレビューお願いします」と伝える
3. ユーザーがレビュー完了してサーバーを閉じたら、TaskOutputでコメントを確認
4. コメント内容に基づいて修正を行う
5. 必要に応じて再度 `difit` で確認

## Constraints

- Gitリポジトリ内でのみ使用可能
- `--pr` モードにはGitHub CLI (`gh`) の認証が必要
